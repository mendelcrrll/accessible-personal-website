Accessible Accordion

WCAG (Web Content Accessibility Guidelines) are guidelines for making web content more accessible to people with disabilities.

Use Tab to navigate between buttons. Use Enter or Space to activate a button. Use arrow keys within accordions for navigation.

Use automated tools like axe, WAVE, or Lighthouse. Also perform manual testing with keyboard navigation and screen readers like NVDA or JAWS.

Accessibility Testing

Manual Testing Checklist

  • Navigate entire page using Tab and Shift+Tab
  • Verify focus indicator is visible on all interactive elements
  • Check that focus order is logical and intuitive
  • Use browser DevTools to inspect ARIA attributes
  • Test with screen reader software (NVDA, JAWS, VoiceOver)
  • Zoom page to 200% and verify layout remains usable
  • Check color contrast with a tool like WebAIM Contrast Checker

Automated Testing Tools

  • axe DevTools - Browser extension for accessibility testing
  • WAVE - Web accessibility evaluation tool
  • Lighthouse - Built into Chrome DevTools
  • NVDA - Free screen reader for Windows
  • WebAIM Contrast Checker - Color contrast validation
  • Keyboard Navigation Tester - Test keyboard navigation flows
